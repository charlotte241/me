# PBW Command Centre — live website on GitHub Pages

The October 2026 Manchester campaign plan + live Ticket Tailor data (pace tracker,
all-events overview, attendees, purchase log) as a self-updating website.

**How it stays live:** a GitHub Action runs every 6 hours, pulls all events and
orders from Ticket Tailor (API key stored as a GitHub secret, never exposed),
writes `docs/data.json`, and GitHub Pages serves the site from `docs/`.

## Deploy with Claude Code (5 minutes)

1. Unzip this folder somewhere on your computer.
2. Open a terminal in the folder and run `claude`.
3. Paste this prompt:

   > Create a private GitHub repo called `pbw-command-centre` from this folder and
   > push everything including the .github directory. Then enable GitHub Pages
   > serving from the /docs folder on main. Tell me the URL when done.

4. Claude Code will use `gh` to do it (log in with `gh auth login` if asked).
5. One manual step Claude Code can't do — add the secret:
   Repo → Settings → Secrets and variables → Actions → New repository secret
   Name: `TICKET_TAILOR_API_KEY` — Value: your key
   (Ticket Tailor → Box office settings → API → create a read key)
6. Repo → Actions → "Update dashboard data from Ticket Tailor" → Run workflow.
   ~1 minute later the site is live and refreshes itself every 6 hours.

## Deploy manually (no Claude Code)

Create a private repo on github.com → upload this folder's contents (keep the
`.github` folder) → add the secret as above → Settings → Pages → deploy from
branch `main`, folder `/docs` → Actions → run the workflow once.

## Privacy — read before sharing the URL

The Pages URL is publicly reachable even on a private repo. Buyer **emails are
excluded by default** (people are matched across events by an anonymous hash, so
NEW/Returning and loyalty tiers still work; names still show). To include emails,
set `INCLUDE_EMAILS: "true"` in `.github/workflows/update-data.yml` — only do
this behind access protection (e.g. Cloudflare Access).

## Files

| File | Purpose |
|---|---|
| `docs/index.html` | The command centre (plan + pace + events + attendees + log) |
| `docs/data.json` | Auto-generated by the workflow — never edit |
| `scripts/fetch_data.py` | Pulls from Ticket Tailor API |
| `.github/workflows/update-data.yml` | Schedule (every 6h) + secret wiring |

Checklist ticks save in each viewer's browser (localStorage), so Adam's ticks
and yours are independent. Change the cron in the workflow for hourly updates.
